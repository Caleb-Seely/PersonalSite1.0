C Compiler

Spring 2020

To better comprehend how the C compiler I use everyday works, I built my own version. The project is well over four thousand lines of code that I have written. There are also numerous other files with code that was used to assist this project. Below you will find just the smallest summary of what the project looked like.

Semantic analysis

After I built the tree, I used it for type checking and other context errors. Some debugging features were also added. There are several flags that can be set to trigger a visual print of the tree with or without types and an explicit grammar walkthrough.
The visual tree code is here.
Semantic analysis is here.

Memory allocation

Once a tree is confirmed to have no erros, the program begins assigning memory usage for each node. Most nodes used one spot in memory but arrays used one more than their given size. Then I built frames around each function for its place in memory. Now to generate the code that was going to be read by a machine, I just needed to look at the memory sizes, their offsets and provide the proper sequence of LD, ADD, ST and backpatches.
Some of that code can be found here.

This was my first large project. A deeper understanign of each part would have made for more efficient code, especially in my error checking and analysis. The code colud also use more comments in some places but the size of the project forced me to better that skill while I was working on it. A majority of this project was new concepts for me. I had never written extensive tree code linked from flex and bison and I had done little assembly code. I was able to get through most of my issues (grammar rules, regular expression oddities and ID types to name a few) leading to a working product but it is not everything I planned for. The final code has trouble with arrays in particual and a range of obvious edge cases. I am happy with what I have learned from it and have no plans to fix those bugs in the near future.

Schedule Planner

Spring 2020

This project uses a database to suggest classes for a computer science student based on their career intrests. New users add themsleves to the database with the add keyword followed by their name, student ID and industy they are intrested in. They can use keywords to add classes they have taken and get semester specific class recomendations based on previously taken courses. At any point a user can return to the page and place their ID in the corner to continue using the software. They can also access the list of all classes as well as the classes they have completed.

The core of this project is written in PHP but it has some HTML and CSS, I also used XAMP, Apache and MySQL. The code is here.

C++

Maze Solution

Spring 2018

This project is early practice of a stack data structure. The program reads in a maze file built from a matrix of periods represnting open cells, hashes for closed cells and S for start G for goal. The c++ program then pushes available cells on a stack and pushes coordnates off when it runs out of cells to push. The program finishes when it finds its goal or runs out of possible cells.


This program uses breadth first search and prioritizes down left. The code could be adjusted to follow a depth first approach but I do not find it necessary.


Additions

I have returned to this code of several occasions to play with it. I first added a print function within the solution loop this allows for a visible walkthrough of the algorithm. It was difficult to see the visited cells so I added colors for a stronger visual. The last addition I made was an alias to run the three above mazes.
